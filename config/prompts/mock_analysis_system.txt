Q3. How is the sink called in the source code?
    Read the line(s) where the sink is invoked.
    Choose ONE:

      [A] standalone
          Direct call: result = sink(arg) or sink(arg)
          → q3_mock_type = "standalone"

      [B] context_manager
          Called inside a with block: with sink(arg) as VAR:
          The mock needs __enter__ and __exit__ methods.
          → q3_mock_type = "context_manager"
          → q5_context_var = the variable name after "as"

      [C] chained
          Call result is immediately chained: sink(arg).method()
          → q3_mock_type = "chained"
          → q3_chained_method = the method name (e.g. "json")
          → q3_chained_depth = how many .method() calls after sink

      [D] standalone_with_close
          Result is assigned, then .close() is called later:
              obj = sink(arg)
              ... obj.close()
          → q3_mock_type = "standalone"  (same mock type as [A])
          → q5_methods_called must include "close"

      [E] return_value
          No mock needed. The target function's return value is inspected
          directly. Choose this when Q1 is null OR when the classification
          strategy is return_value or filesystem_state.
          → q3_mock_type = null
          → All Q4, Q5, Q8 fields become null.
